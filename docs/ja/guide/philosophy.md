# 理念

**意思決定を減らし、より良いインターフェースを。**

Pomikit は、プレミアムな UI を標準で提供しつつ、開発者に求める意思決定を最小にします。デザインは一度決め、コンポーネントはトークンを消費し、ビジネス意図から挙動を推論します。

## 意思決定フィルタ

prop や機能を追加する前に、こう問いかけてください：

> これは開発者に追加の意思決定を求めるか？

**はい** なら、次の順で試します：

1. Design Kit / tokens（アプリを一度でテーマ化）
2. 意図推論（非同期クリック、`pending`、`options`、`error`）
3. 慣例的な挙動（妥当なデフォルト）

その上で初めて prop を公開し、しかもそれが**ビジネス意図**を表す場合に限ります。

## 意図 vs 美学

| 優先する（意図） | 製品 API として避ける（美学） |
| --- | --- |
| `confirm`、`href`、`required` | `variant`、`tone`、`size` |
| `pending`、`error`、`options` | コンポーネント上の `radius`、`density` |
| 非同期クリックハンドラ | インスタンスごとの色 / 影のつまみ |

スタイル prop は高度な逃げ道として残しても構いません。それ自体が製品ではありません：はじめに、プレイグラウンド、主要な例には出さないでください。

## 一度でデザイン

```ts
createTheme({ design: 'linear', accent: '#5B5FFF' })
```

Design Kits が色、角丸、影、余白、タイポグラフィ、モーション、サーフェスを担います。コンポーネントは `design === …` で分岐せず、CSS トークンを読みます。

## ドキュメントの立場

最小 API から導きます。Storybook の逃げ道ストーリーは内部の隔離ツールです。本サイトが公式の面です。
