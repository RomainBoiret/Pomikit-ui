# 철학

**결정 더 적게. 인터페이스는 더 좋게.**

Pomikit는 기본적으로 프리미엄 UI를 제공하면서 개발자에게 요구하는 결정을 최소로 합니다. 디자인은 한 번 정하고, 컴포넌트는 토큰을 소비하며 비즈니스 의도에서 동작을 추론합니다.

## 결정 필터

prop이나 기능을 추가하기 전에 물어보세요:

> 이것이 개발자에게 추가 결정을 요구하는가?

**예**라면 다음 순서로 시도하세요:

1. Design Kit / tokens (앱을 한 번에 테마화)
2. 의도 추론 (비동기 클릭, `pending`, `options`, `error`)
3. 관례적 동작 (합리적인 기본값)

그다음에야 prop을 노출하고, **비즈니스 의도**를 표현할 때만 가능합니다.

## 의도 vs 미학

| 선호 (의도) | 제품 API로 피할 것 (미학) |
| --- | --- |
| `confirm`, `href`, `required` | `variant`, `tone`, `size` |
| `pending`, `error`, `options` | 컴포넌트의 `radius`, `density` |
| 비동기 클릭 핸들러 | 인스턴스별 색 / 그림자 조절 |

스타일 prop은 고급 탈출구로 남을 수 있습니다. 그것이 제품은 아닙니다: 시작하기, 플레이그라운드, 주요 예시에는 넣지 마세요.

## 한 번만 디자인

```ts
createTheme({ design: 'linear', accent: '#5B5FFF' })
```

Design Kits가 색, 둥글기, 그림자, 간격, 타이포그래피, 모션, 서피스를 소유합니다. 컴포넌트는 `design === …`로 분기하지 않고 CSS 토큰을 읽습니다.

## 문서 입장

최소 API로 안내합니다. Storybook 탈출구 스토리는 내부 격리 도구입니다. 이 문서 사이트가 공식 표면입니다.
